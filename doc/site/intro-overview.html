1.1. Overview

ZSCRIPT is a modern, dynamically typed scripting language designed for flexibility, ease of use, and powerful runtime features. It offers a hybrid type system, combining the best of both weak and strong typing, giving developers the freedom to choose between dynamic flexibility and static type safety when needed.

ZSCRIPT is inspired by several well-known scripting languages but introduces unique elements that make it adaptable to various use cases, from rapid prototyping to more structured application development. Its simple syntax and diverse set of features make it suitable for tasks ranging from simple scripts to complex systems with performance requirements.

Key Features

  • Hybrid Typing System: ZSCRIPT allows variables to be declared either weakly typed or strongly typed, offering the flexibility of dynamic typing while enabling stricter type enforcement where necessary. Developers can choose to rely on weak typing for rapid development and experimentation or enforce strong typing to prevent errors and ensure predictable behavior.
    • Weak Typing: Variables declared with the var keyword can hold values of any type and can be reassigned to different types throughout their lifetime.
    • Strong Typing: Variables can be declared with explicit types like int, float, or string. Once declared, they are restricted to those types, preventing accidental type errors.
  • Type Inference and Auto Typing: ZSCRIPT introduces an auto keyword that automatically infers the type of a variable based on its initial value. This combines the flexibility of dynamic typing with the safety of strong typing, without requiring the developer to explicitly specify the type.
  • Type Restrictions: Developers can restrict variables to specific sets of types. For example, a variable can be declared to only accept int or float, providing both flexibility and control. ZSCRIPT also includes convenient shortcuts, like the number keyword, which is equivalent to var<int, float>, simplifying code while maintaining type restrictions.
  • Constant Variables: Immutability is supported through the const keyword, allowing developers to define constant values that cannot be changed after initialization. This is useful for defining configuration values, constants, or any data that should remain fixed throughout the execution of a script. ZSCRIPT enforces that constant variables be initialized with literal values, enhancing performance and predictability.
  • Global Variables: ZSCRIPT supports global variables, which can be declared and accessed from anywhere within a script. By using the global keyword, developers can clearly indicate that a variable is intended to be globally accessible, ensuring clarity and avoiding accidental overwrites of local variables.
  • Simple and Powerful Syntax: ZSCRIPT has a clean and intuitive syntax that lowers the barrier to entry while still providing powerful features for experienced developers. The language offers constructs like arrays, tables (dictionaries), functions, classes, and instances, making it versatile and capable of handling complex scenarios.
  • Extendability and Interoperability: With types like user_data and raw_pointer, ZSCRIPT is designed to interact with external systems and libraries, making it suitable for embedding in larger applications, game engines, or systems that need customizable scripting capabilities. This enables integration with C/C++ libraries and other environments.

Why ZSCRIPT?

The primary motivation behind ZSCRIPT is to provide developers with a language that strikes a balance between dynamic flexibility and static reliability. Many modern scripting languages, while powerful, either impose too much structure or too little, leaving developers to choose between type safety and ease of use. ZSCRIPT’s hybrid approach allows developers to move fluidly between the two extremes as needed.

  • Flexibility for Beginners: Beginners can start with weak typing and basic constructs without worrying about strict rules, while gradually adopting more advanced features as they become comfortable.
  • Power for Experts: Advanced users can take full advantage of the strong typing system, type inference, and type restrictions to write efficient, safe, and maintainable code, especially in larger projects or performance-critical applications.

Use Cases

ZSCRIPT is designed to be highly adaptable to a wide range of use cases, including but not limited to:

  • Game Scripting: The language’s hybrid typing system allows for the fast-paced, iterative development cycles required in game scripting. ZSCRIPT’s support for arrays, tables, and user-defined classes make it an excellent choice for game logic, AI behaviors, and event handling.
  • Embedded Systems: With its lightweight footprint and dynamic nature, ZSCRIPT can be embedded into larger applications, providing a scripting interface for customization or automation without needing to rewrite or recompile the entire system.
  • Prototyping: ZSCRIPT’s weak typing and flexible syntax make it ideal for rapid prototyping, allowing developers to experiment with ideas and features quickly before committing to more structured code.
  • Tooling and Automation: ZSCRIPT can be used for scripting tools, automating workflows, or creating small applications that require a combination of flexibility and power. Its mix of dynamic and strong typing enables quick iteration while maintaining reliability where needed.
